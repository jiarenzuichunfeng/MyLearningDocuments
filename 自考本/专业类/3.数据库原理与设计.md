# 数据库原理与设计
## 数据库基本概念
* 数据
	* 数据是描述事物的符号记录，是指利用物理符号记录下来的，可以鉴别的信息
* 数据库（DBS）
	* 数据库是指长期存储在计算机中有组织，可共享的数据集合
	* 数据要按照一定的数据模型组织、描述和存储，具有较小的冗余度、较高的数据独立性，系统易于扩展，并可以被多个用户分享。
	* 数据库中存储数据的三个基本特点
		* 永久存储
		* 有组织
		* 可共享
* 数据库管理系统（DBMS）
	* 是专门用于建立和管理数据库的一套软件，介于软件应用程序于操作系统之间
	* 功能
		* 数据定义
		* 数据操作
		* 数据库运行管理
		* 数据库建立和维护
		* 数据组织、存储和管理
		* 其他功能
* 数据库系统的构成
	* 用户 - 应用程序 - 数据库管理系统（数据库管理员（DBA））- 数据库
* 数据库管理技术发展
	* 人工管理
	* 文件系统（物理唯一性）
	* 数据库系统阶段（逻辑）
		* 数据集成
		* 数据共享性高
		* 数据冗余小
		* 数据一致性
		* 数据独立性高
		* 实施统一管理于控制
			* 安全性
			* 完整性
			* 并发控制于故障恢复
* 数据库系统的结构分类
	* DBA视角
		* 内部系统结构，采用三级模式
			* 模式（概念或逻辑）（概念试图）
			* 内模式（存储模式）（内部视图或存储视图）
			* 外模式（子模式或用户模式）（用户视图或数据视图）
		* 外部系统结构
			* 集中式
			* 分布式
			* 并行结构
	* 用户视角
		* 客户/服务器结构
		* 浏览器/服务器结构
	* 三级模式结构的两层映像与数据独立性
		* 映像，就是一种对应规则，他指出映像双方是如何进行转换的
		* 外/模 逻辑
		* 内/模 物理
	* 客户/服务器结构
		* 客户/服务器结构（C/S）
			* 表示层——服务层
		* 浏览器/服务器结构（B/S）
			* 表示层——处理层——服务层
* 数据模型
	* 模型
		* 现实世界特赠的模拟和抽象表达
	* 数据模型
		* 现实世界数据特征的抽象，描述的是数据的共性内容
		* 数据模型组成要素
			* 数据结构——描述系统的静态特征
			* 数据操作——描述系统的动态特征
			* 数据约束——描述数据结构中数据间的语法和语义关联
		* 数据模型是模型化数据和信息的工具，也是数据库系统的核心和基础
		* 概念层数据模型（概念模型或信息模型）
			* 概念层是数据抽象级别的最高层。主要用于数据库的设计阶段
		* 数据逻辑模型（中间层）
			* 任何数据库管理系统都是基于某种逻辑模型
			* 层次模型
				* 最早使用的一种数据模型
				* 有且仅有一个节点，没有父节点，称作根节点
				* 其他节点有且仅有一个父节点
			* 网状模型
				* 以网状结构表示实体与实体间的联系
				* 允许结点有多于一个父节点
				* 可以有一个以上的节点没有父节点
			* 关系模型
				* 用二维表结构来表示实体间的联系
				* 建立在严格的数据概念的技术上
				* 概念单一
				* 存取路径对用户透明，有更高的数据独立型，更好的安全保密性
			* 面向对象模型
				* 即使概念模型又是逻辑模型
				* 表达能力丰富，对象可复用、维护方便
		* 物理模型
			* 物理层数据模型，也称为数据的物理模型，描述数据在存储介质上的组织结构，是逻辑模型的物理实现
			* 是数据库最底层的抽象
			* 设计目标是提高数据库性能和有效利用存储空间
	* 数据特征
		* 静态特征
			* 数据的基本结构
			* 数据间的联系
			*  数据取值范围的约束
		* 动态特征
			* 对数据可以进行符合一定规则的操作
	* 信息世界涉及的基本概念
		* 实体（Entity）
		* 属性（Attribute）
		* 码或键（key）
			* 属性（或属性组）的值，都能用来唯一标识该关系的元组
		* 超码或超健（Super Key）
			* 在码中去除某个属性，它仍然是这个关系的码
		* 候选码或候选键（Candidate Key）
			* 在码中不能从中移去任何一个属性，否则它就不再是这个关系的码或建
			* 候选码或候选键是这个关系的最小超码或超键
		* 主属性或非主属性
			* 包含在任何一个候选码中的属性被称为主属性或码属性
		* 主码或主键
			* 在若干个候选码中指定一个唯一标识关系的元组（行）
		* 全码或全键
			* 一个关系模式的所有属性集合是这个关系的主码或主键
		* 外码或外键
			* 某个属性或属性组不是这个关系的主码或候选码，而是另一个关系的主码
			* 参照关系也成为从关系，被参照关系也称为主关系，它们是指以外码相关联的两个关系
		* 域（Domain）
			* 表示属性的取值范围
		* 数据类型
			* 每个列都有它的数据类型
		* 关系模式
			* 关系模式是型，关系是值
		* 实体型（EntityType）
		* 实体集（Entity Set）
		* 联系（Relationship）
* 关系数据库
	* 关系数据模型的组成要素
		* 关系数据结构
			* 关系数据库对关系的限定/要求
				* 每一个属性都是不可被分解的（不允许表中有表）
				* 每一个关系仅仅有一种关系模式
				* 每一个关系模式中的属性必须命名，属性名不同
				* 同一个关系中不允许出现候选码或候选键值完全相同的元组
				* 在关系中元组的顺序是无关紧要的，可以任意交换
				* 在关系中属性的顺序是无关紧要的，可以任意交换
		* 关系操作集合
		* 关系完整性约束
			* 数据库的数据完整性是指数据库中数据的正确性、相容性、一致性
			* 实体完整性
				* 主码不能为空
			* 参照完整性
				* 要么等于主码的内容要么为空
	* 关系数据库的规范化理论
		* 关系模式中可能存在的冗余和异常问题
		* 函数依赖与关键字
			* 完全函数依赖
				* 两个及以上属性可以找到一个属性
			* 部分函数依赖
				* 一个属性可以找到一个属性
			* 传递函数依赖
				* 有一个属性1可以找到属性2但不可以找到属性3，但属性2可以找到属性3
		* 范式与关系规范化过程
			* 第一范式1NF（默认的excle表）
			* 第二范式2NF 在第一范式的基础上抽从表
			* 第三范式3NF 在第二范式的基础上消除传递函数依赖
			* 第三范式的改进形式 
				* 特殊情况（第二范式有传递函数依赖，但所有属性都是注码）还是消除传递函数依赖
